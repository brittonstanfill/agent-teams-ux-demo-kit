Why this isn't a tab bar

The current flow buries hotel and meal vouchers in an FAQ called "Other policies." That's the design saying "we'd prefer you didn't ask." Pushing entitlements up to Screen 02 trades a few pixels of polish for a measurable drop in calls and out-of-pocket spend by tired travelers. The stack reads top-to-bottom in screen-reader order: done · done · your turn.

Why "Confirmed when you tap"

The brief flags that the current Screen 2 makes standby look like a confirmed rebooking. Held seats are honest about their state — you only own them after the tap. We named that out loud rather than relying on a tab color. Two cards instead of three; "Recommended" is replaced with a real comparison ("Closest to your original arrival" vs. "Nonstop, slower arrival") so the user can pick on their goal, not ours.

Why the call button never disappears

Reducing support calls by hiding the support number is the dark-pattern version of this goal. We kept the call action visible on every screen — but pair it with a wait estimate and a "kids/mobility" cue so users self-route to the channel that's actually better for them.

Accessibility calls we made

  • All interactive targets ≥ 44×44 (WCAG 2.5.5).
  • Body text contrast ≥ 4.5:1, large text ≥ 3:1 (WCAG 1.4.3); checked in both schemes.
  • Status is never color-alone — every chip carries text (WCAG 1.4.1).
  • Focus rings visible on all interactive elements (WCAG 2.4.7).
  • Reduced-motion respected (WCAG 2.3.3).
  • Care tiles use semantic article/section/ol so screen readers announce the plan as a list, not a soup of divs (WCAG 1.3.1).
