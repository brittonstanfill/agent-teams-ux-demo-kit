When a flight gets canceled at 10:46 p.m., plain language and visible help beat polished apology.

Five screens for a tired, mobile-only traveller trying to reach a family event tomorrow. The flow names what happened, surfaces support that is already owed, and never makes the disrupted trip feel like a sales surface.

01 · AlertSay the real thing, in the first message.

The text the traveller wakes up to has to do three jobs: name what happened, hold their seat, and signal that help is real. Not "schedule irregularity."

  • Copy"Canceled" not "schedule irregularity." Plain language is the first trust signal.
  • TrustA seat is pre-held on the next viable option, with a visible hold timer. Loss aversion is engaged honestly — they're not being rushed, they're being held.
  • FixHotel and meal support are surfaced in the first message, not buried in an FAQ.
  • A11ySMS reads cleanly in screen-reader linear order; the link uses a short, brandable code so it survives line wrapping and Braille displays.

02 · What & choicesThree roads. No buried defaults.

The current flow opens on a vague itinerary change and defaults to travel credit. We open on the real event and present every meaningful path at the same depth — including a path to a human.

  • FixRemoves the "Continue" button that didn't say what would happen. Each card states its consequence in 6–10 words.
  • FixNo default tab forcing travel credit. Rebook is featured because the user goal is "get there," but refund and human are one tap, same surface.
  • TrustEntitlements banner is visible before the user picks an option. People shouldn't have to commit to a path to see what's owed.
  • A11yChoice list uses semantic list/listitem roles. Icons are decorative (aria-hidden); labels carry the meaning. Targets ≥ 44×44.

03 · Pick a flightSort by what they care about, not what you score.

"Recommended" is opaque. Replace it with a reason the user can verify. Sort by arrival time by default, because the user goal is to land — not to depart.

  • Copy"Closest to your original arrival" replaces "Recommended." It's checkable; the user knows why this is first.
  • FixNo fare-difference price tag during a disruption the carrier caused. Pricing belongs in a buy flow, not a recovery flow.
  • FixFilters cover real goals (nonstop, arrive AM, family-friendly, mobility help) — not airline-coded categories.
  • TrustStandby is visible but visually demoted and labeled as a chance, not a confirmed seat.

04 · Overnight supportIf it's owed, it shouldn't be hidden in an FAQ.

Hotel and meal credit were inside a collapsed "Other policies" panel in the current flow. That's a low-trust pattern under stress. Here, what's available is named, opt-out by default, and reversible.

  • FixEligible support is opt-out, not buried. The user can always say "skip — I'll handle it myself" with one tap.
  • Trust"On" / "Off" replace marketing language. No "complimentary" or "courtesy" — the system shows status, not framing.
  • A11yToggles have visible labels (not just colour). Each switch announces its state to screen readers; touch target wraps the full row.
  • Copy"Travel-with help" is named explicitly — children, mobility, companion — and routes to a specialist, not a generic queue.

05 · You're setEnd with a useful summary, not a "Done" button.

The current confirmation says "Your changes have been applied" and offers a Done button. That's a deflection screen, not a confidence screen. Replace it with everything the user needs to walk out the door tomorrow morning.

  • FixSummary names the flight, the hotel, the meal credit, baggage status, and check-in moment — in that order of "what do I do next."
  • TrustAlways-visible "if plans change" link returns to options without restarting the flow. People know they can re-pick.
  • A11y"Save offline" downloads a PDF and adds to the OS wallet. Works at low bandwidth and inside the screen-reader linear flow.
  • CopyNo exclamation marks, no "all done!" — fatigue and stress respond better to calm specifics than to celebration.
