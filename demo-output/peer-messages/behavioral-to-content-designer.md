Subject: Ethical framing / phrasing risks in the canceled-flight recovery flow

Hi —

Handing you the behavioral guardrails on copy. I'm not writing the words; I'm naming the cognitive moments the copy has to land in. Push back wherever you think the framing is wrong for the voice.

## The framing risks I'm flagging

### 1. Loss-aversion language — use it only when factually true
- The current flow buries entitlements (hotel, meal). If copy says "Don't miss your hotel voucher" when the user is genuinely eligible — that's an honest loss frame. Keep.
- If copy says "Don't lose your spot — book now" with no real inventory pressure, that's manipulated urgency. Drop.
- **Behavioral spec:** loss framings are ethical when (a) the loss is real, (b) the user can verify it, and (c) the framing is in service of the user's stated goal — getting home. Use that test on every "don't miss / don't lose / last chance" line.

### 2. "Recommended" — replace with a labeled reason
- Unexplained authority labels trigger reactance in stressed users. They suspect they're being steered (often correctly).
- **What the copy must accomplish behaviorally:** in one short line, tell the user *why* this option is recommended for *them*. Examples of the shape (not final copy):
  - "Earliest arrival in your original window"
  - "Same arrival airport, fewest changes"
  - "Closest match to your original itinerary"
- **Test for whether the label is earned:** if you can't write the reason in one short line, the label isn't earned — drop it.

### 3. Urgency / countdown copy — only when user-initiated
- Avoid: "3 seats left at this price," "act in the next 5 minutes," any countdown that started without the user asking for a hold.
- OK: a visible hold timer *after* the user has tapped "Hold this seat" — because that's a state the user themselves created.
- **Behavioral spec:** scarcity cues are ethical when they describe a state the user opted into; they become dark patterns when the system invents pressure to push a decision.

### 4. Fare-difference framing
- "$0" / "+$84" sets an implicit zero anchor against a disruption the user didn't cause. Loss aversion kicks the user toward the $0 option even when it's a worse fit (later arrival, more stops).
- **What the copy must accomplish:** make the fare-difference comparable to the *original ticket the user already paid for*, not to the cheapest recovery option. Something like "Same fare as your original ticket" / "+$84 vs. your original" — phrased your way.
- If fare difference is $0, don't say "$0" prominently — say "same fare" or just don't add a price chip. The absence of price is the absence of price.

### 5. SMS opener — "Schedule irregularity" is an evasion
- The current SMS reads as a euphemism. Erodes trust before the app opens.
- **What the copy must accomplish:** plain language ("Your 6:15 a.m. flight NS482 has been canceled."), one reassurance, one next step. The behavioral job here is to convert anxiety + ambiguity into a concrete next action — ambiguity aversion otherwise drives them to call before the app even loads.

### 6. Confirmation screen — peak-end rule
- "Trip updated / Your changes have been applied / Done" is the worst possible ending for a stressful disruption. The peak-end rule says the ending disproportionately shapes the user's memory of the whole airline.
- **What the copy must accomplish on confirmation:** concrete recap (new flight time, gate if known, hotel status, baggage status, check-in info, what to do if this changes again). Calm tone. No marketing. This is where trust gets rebuilt for the next trip.

### 7. Decline copy — no confirmshaming
- When the user declines a hotel voucher or a recommended flight, do not write "No thanks, I'll figure it out myself" or any guilt-laden variant.
- **Behavioral spec:** decline labels should be neutral and equal-weight with accept labels.
