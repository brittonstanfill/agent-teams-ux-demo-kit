# Meeting One-Pager: Lens Relay Demo

## Claim

Agent teams are useful when the answer improves through coordinated specialist disagreement, not when we simply want more output.

## Demo

Use a fictional canceled-flight recovery flow. One traveler is stressed, mobile-only, and trying to recover from a 10:46 p.m. flight cancellation.

## Team

- UX Researcher: user reality and assumptions.
- Information Architect: flow and structure.
- Interaction Designer: states and recovery paths.
- Content Designer / UX Writer: language and tone.
- Accessibility Specialist: blockers and inclusive guardrails.
- Visual / UI Designer: hierarchy and component direction.
- Behavioral Scientist: trust, defaults, and ethical choice architecture.

## System

1. **Lens:** each role inspects the same artifact independently.
2. **Relay:** each role must message at least two other roles.
3. **Fuse:** the lead resolves conflicts into one recommendation.
4. **Score:** compare against a single-agent baseline.

## What The Audience Should See

- A shared task list.
- Separate role-owned outputs.
- Direct handoffs between teammates.
- At least one conflict or blocker before synthesis.
- A final recommendation that names what changed because agents communicated.

## The Practical Rule

Use the lowest-coordination primitive that preserves quality.

- Skill: repeatable know-how.
- Subagent: focused worker returns a result.
- Agent team: teammates need to talk, challenge, and coordinate.
- Managed agent: recurring, hosted, credentialed work.

## Success Criteria

The team earns its cost only if it catches more severe issues, resolves real tradeoffs, improves accessibility/trust, or creates a better experiment plan than the single-agent baseline.
